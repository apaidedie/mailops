# Overview command center visual polish

## Goal

Make the dashboard overview feel like a professional operational command center for a unified mailbox aggregation service instead of a decorative glassmorphism preview panel.

## User Value

Operators should understand service health, account inventory, provider activity, external API usage, mailbox pool state, and recent activity without decorative labels, structural emoji, or explanatory hover copy competing with the data.

## Confirmed Facts

The project uses Flask templates with vanilla CSS and JavaScript for this surface. No React, Tailwind, or component library is detected. The overview first viewport currently contains decorative copy such as `玻璃态概览面板` and `细腻卡片视图`, structural emoji in the title and tab labels, and hover-note overlays generated by `renderHoverNote()`. The UI direction from the local design skills is an operational SaaS/data-dense dashboard: dense but calm, restrained color, sharp hierarchy, token-driven surfaces, and no emoji as structural icons.

## Requirements

The overview header must use operational product copy and must not mention glassmorphism or refined card styling. The overview title and primary tabs must not use emoji as structural icons. Cards and timeline rows must stop rendering explanatory hover-note overlays. The overview CSS must use calmer token-based surfaces, smaller radii, restrained shadows, accessible focus states, and responsive wrapping without decorative orbs/blobs. Existing overview data loading, tab switching, i18n behavior, API payload handling, and backend routes must remain compatible.

## Acceptance Criteria

- [x] The overview HTML and JavaScript no longer render `玻璃态概览面板`, `细腻卡片视图`, `📊 数据概览`, or emoji-prefixed overview tab labels.
- [x] `static/js/features/overview.js` no longer emits `.ov-hover-note` overlays or card-level explanatory hover text.
- [x] Overview card and shell CSS no longer depends on decorative pseudo-element orbs for the first viewport and keeps the dashboard visually restrained.
- [x] Frontend contract tests cover the professionalized overview copy, structural emoji removal, hover-note removal, and key DOM hooks.
- [x] Existing overview and frontend contract tests pass.

## Out of Scope

Backend overview metrics, provider contracts, external API behavior, mailbox provider routing, and non-overview navigation surfaces are not changed in this task.
